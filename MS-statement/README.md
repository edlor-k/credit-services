# Statement Microservice (MS-statement)

## Описание

**MS-statement** — микросервис для обработки анкет клиентов и получения кредитных предложений.  
Выполняет прескоринг заявки, обращается к внешнему сервису сделок (MS-deal) для получения и выбора кредитных предложений.

---

## Описание API

- **POST `/statement`**  
  Получение списка кредитных предложений по анкете клиента.  
  **Request:** `LoanStatementRequestDto`  
  **Response:** `List<LoanOfferDto>`

- **POST `/statement/offer`**  
  Выбор одного из кредитных предложений для заявки.  
  **Request:** `LoanOfferDto`  
  **Response:** `void`

---

## Стек технологий

- **Java 21**
- **Spring Boot 3**
- **Lombok**
- **Jakarta Validation** — валидация данных запроса
- **SLF4J** — логирование
- **Swagger** (springdoc-openapi) — документация API
- **JUnit 5 + Mockito** — тестирование
- **MapStruct** — маппинг DTO/Entity

---

## Структура проекта

- **/dto** — DTO-запросы и ответы
- **/model/entity** — сущности для прескоринга
- **/model/enums** — перечисления ошибок и статусов
- **/mapper** — MapStruct-мапперы
- **/service** — интерфейсы бизнес-логики и интеграций
- **/service/impl** — реализации бизнес-логики
- **/exception** — кастомные исключения и обработка ошибок
- **/util** — утилиты и константы (валидация, сообщения об ошибках)
- **/config** — конфигурация приложения (Swagger, RestClient, properties)

## Работа с ошибками

- Все ошибки возвращаются в структурированном формате (`ErrorResponseDto`) с кодом и подробностями.
- Поддерживается единый стиль обработки ошибок: валидация, бизнес-ошибки, ошибки интеграции.
- Ошибки логируются и доступны для анализа.

---

## Взаимодействие с внешними сервисами

- Интеграция с микросервисом сделок (**MS-deal**) для получения и выбора кредитных предложений.
- Все вызовы внешних сервисов инкапсулированы в отдельном сервисе с обработкой ошибок и логированием.
- Настройки интеграции (`base-url`, пути) вынесены в файл конфигурации (`application.yml`).
