# Calculator Microservice

## Описание

**Calculator** — это микросервис кредитного калькулятора, предназначенный для расчёта кредитных предложений, 
скоринга и расчёта кредитных параметров по заявке пользователя.

---

## Описание API

- **POST `/calculator/offers`** — расчет возможных кредитных предложений (прескоринг, 4 комбинации условий).
- **POST `/calculator/calc`** — скоринг клиента и расчет финальных параметров кредита (ежемесячный платёж, ПСК, график платежей).

---

## Стек технологий

- **Java 21**
- **Spring Boot 3**
- **Lombok**
- **Jakarta Validation**
- **SLF4J** для логирования
- **Swagger** (springdoc-openapi) — документация API
- **JUnit 5 + Mockito** — тестирование

---

## Структура проекта

- **/dto** — структуры запросов и ответов
- **/model/enums** — типизированные данные (пол, статус и т.д.)
- **/service/business/prescoring** — логика прескоринга, генерация кредитных предложений
- **/service/business/scoring** — логика скоринга, расчёт параметров кредита
- **/service/business/schedule** — расчёт графика платежей
- **/util** — используемые утилиты и константы
- **/exception** — кастомные исключения и обработка ошибок
- **/config** — конфиг приложения

## API

### 1. Прескоринг — расчет предложений

**POST** `/calculator/offers`  
Request: `LoanStatementRequestDto`  
Response: `List<LoanOfferDto>`

- На вход: сумма, срок, ФИО, email, дата рождения, паспортные данные
- На выход: 4 предложения с разными опциями (страховка/зарплатный клиент)

### 2. Скоринг и полный расчёт

**POST** `/calculator/calc`  
Request: `ScoringDataDto`  
Response: `CreditDto`

- На вход: подробные данные клиента (анкета, занятость, опции)
- На выход: параметры кредита (ставка, ПСК, ежемесячный платёж, график платежей)

> Протестировать API можно через Swagger UI, доступный по адресу:
> [`/swagger-ui.html`](http://localhost:9010/swagger-ui.html)  
> или  
> [`/swagger-ui/index.html`](http://localhost:9010/swagger-ui/index.html)

## Ошибки и бизнес-валидация

- Все ошибки возвращаются в читабельном виде (на русском языке).
- Прескоринг и скоринг реализованы отдельно, все причины отказа описаны в константах и логируются.
