# Deal Microservice (MS-deal)

## Описание

**MS-deal** — микросервис для управления кредитными заявками.  
Обеспечивает полный цикл работы с заявкой: создание, генерация кредитных предложений, выбор предложения, расчет и сохранение финальных параметров кредита.

---

## Описание API

- **POST `/deal/statement`**  
  Создание кредитной заявки, запрос предложений у микросервиса калькулятора (MS-calculator).  
  **Request:** `LoanStatementRequestDto`  
  **Response:** `List<LoanOfferDto>`

- **POST `/deal/offer/select`**  
  Выбор кредитного предложения для заявки.  
  **Request:** `LoanOfferDto`  
  **Response:** `void`

- **POST `/deal/calculate/{statementId}`**  
  Расчет финальных параметров кредита и завершение оформления заявки.  
  **Path variable:** `statementId` — ID заявки  
  **Request:** `FinishRegistrationRequestDto`  
  **Response:** `void`

## Стек технологий

- **Java 21**
- **Spring Boot 3**
- **Lombok**
- **Jakarta Validation**
- **SLF4J** — логирование
- **Swagger** (springdoc-openapi) — документация API
- **JUnit 5 + Mockito** — тестирование
- **PostgreSQL** — база данных
- **Docker** — контейнеризация

---

## Структура проекта

- **/dto** — DTO-запросы и ответы
- **/model/entity** — сущности
- **/model/enums** — перечисления (статусы, роли и др.)
- **/mapper** — MapStruct-мапперы
- **/repository** — Spring Data репозитории (используется JpaRepository)
- **/service** — интерфейсы бизнес-логики
- **/service/impl** — реализации бизнес-логики
- **/util** — утилиты и константы
- **/exception** — кастомные исключения и обработка ошибок
- **/config** — конфигурация приложения

## Работа с ошибками

- Все ошибки возвращаются в структурированном формате с пояснением причины (включая бизнес-валидацию).
- Для ошибок внешнего калькулятора или некорректных данных пользователя возвращается подробное описание с деталями.
- Основные ошибки логируются.

---

## Взаимодействие с внешними сервисами

- Интеграция с микросервисом калькулятора (`MS-calculator`) для расчёта кредитных предложений и параметров кредита.
- Все вызовы внешних сервисов инкапсулированы в специализированных сервисах, реализующих обработку ошибок и парсинг ответов.
